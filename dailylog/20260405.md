## 問題追蹤：DEMO-PLUS 卻顯示「需要 PLUS 方案」

- **現象**：已登入 DEMO-PLUS，PLUS 頁面仍顯示升級提示。  
- **原因**：JWT payload 使用 **`planCode`（camelCase）**；前端 `useSessionMe` 僅讀 **`me.plan_code`（snake_case）** 時會變成 `undefined`，誤判為非 PLUS／PRO。  
- **處理**：於 `useSessionMe` 在寫入 `me` 前呼叫 `normalizeMeDto()`，將 `plan_code` 與 `planCode` 合併為同一值並回寫兩欄位；hook 另回傳 **`planCode`**（與 `me.plan_code` 同步）供頁面使用。  
- **驗證建議**：DevTools → Application → Local Storage 取 JWT，jwt.io 解碼確認 `planCode`；Network → `/auth/me` 確認回應是否為 snake 或 camel（若有 API 閘道改寫鍵名，正規化後應一致）。

### 追加根因（仍顯示需要 PLUS）

- **頂欄「NEXORA Demo PLUS」** 來自租戶 **`name_en`**（例：`seed` 中 `DEMO-PLUS` 租戶），**不是**訂閱方案的技術代碼。  
- **`nx99_plan.code` 實際為 `NEXORA-PLUS` / `NEXORA-LITE` / `NEXORA-PRO`**，JWT 與 `/auth/me` 的 `plan_code` 亦為此格式。  
- 前端與 `PlusPlanGuard` 原先只接受字串 **`PLUS`／`PRO`**，故 **`NEXORA-PLUS` 永遠被判成非 PLUS**。  
- **修正**：共用 `planSupportsNx02PlusFeatures()`，同時接受 `NEXORA-PLUS`、`NEXORA-PRO`、`NEXORA-ENTERPRISE` 與簡寫（nx-api guard + nx-ui 各一檔，邏輯對齊）。
