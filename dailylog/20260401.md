## 遇到的問題與解法

| 問題 | 解法 |
|------|------|
| 明細與列表並排時列表被壓縮 | 明細改 `fixed` 浮層＋遮罩，列表不再與明細同 flex 列。 |
| 明細底部表單被裁切 | 拿掉衝突的固定高度鏈，改整塊明細 `overflow-y-auto`；Tab 內改一般區塊捲動。 |
| `useEffect`（Esc）在 `panelOpen` 定義前引用 | 將 Esc 監聽 **移到** `panelOpen`／`closeDetailFull` 定義之後。 |
| 零件主檔 **`[nxui_nx00_part_list_001] 500`** | 本機 **`nexora_core`** 未套用與 **`schema.prisma`** 同步之 migration（缺 **`nx00_part.sec_code`** 等）。對 **nx-api 所用連線** 執行 **`pnpm exec prisma migrate deploy`**（於 `packages/db-core`）。 |
| 職務篩選開啟時 ↑↓ 同時動到列表 | 僅在 **`dropdown-menu-content[data-state=open]`** 時略過列表快捷鍵；關閉後 **`onCloseAutoFocus` + `preventDefault`** 將焦點移回列表區。 |
| 篩選關閉後方向鍵無法選列 | 焦點留在觸發鈕時行為不一致 → **關閉選單後 `focus` 列表容器**（`tabIndex={-1}`）。 |
