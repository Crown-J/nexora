## 今日完成事項

1. **`nx-ui` 本機建置失敗排查**（`pnpm --filter nx-ui build`）  
   - 現象：Turbopack 回報 `Module not found`，無法解析 `@radix-ui/react-dialog`、`@radix-ui/react-label`、`@radix-ui/react-tabs`。  
   - 原因：`package.json` 與根目錄 `pnpm-lock.yaml` 已宣告上述套件，但 `apps/nx-ui/node_modules/@radix-ui` 實際缺少對應目錄（`node_modules` 與 lockfile **不同步**）。  
   - 處理：於 **monorepo 根目錄**執行 `pnpm install`，補齊連結後建置通過。  
   - 備註：Windows PowerShell 不支援 `&&` 串指令時，可改 `Set-Location <repo>; pnpm --filter nx-ui build`。

2. **Vercel 部署「建置成功但首頁 404」說明與設定修正**  
   - 原因摘要：Next.js 的 `.next` 為**框架內部輸出**，不應在 Vercel 當成靜態站的 **`outputDirectory`**；先前日誌（`20260328.md` 第 12 點）將 `outputDirectory` 設為 `apps/nx-ui/.next` 的作法，易導致部署狀態 Ready、實際路由卻 404。另 **`distDir: ".next"`** 本為 Next 預設，無法單靠它修正 Vercel 行為。  
   - 程式／設定變更：  
     - **`apps/nx-ui/next.config.ts`**：已移除僅等於預設值的 `distDir: ".next"` 及相關註解（先前提交）。  
     - **不在 repo 內放 `apps/nx-ui/vercel.json`**（見下方第 3 點）：改由 Vercel 專案設定處理 monorepo 安裝指令。  
   - Vercel 後台建議再確認：**Root Directory** = `apps/nx-ui`、**Include files outside the root directory** 開啟；**勿**自訂 **Output Directory**；**勿**保留舊版 Production 覆寫（`Framework: Other`、`Output Directory: apps/nx-ui/.next` 等）；若仍異常可 **Redeploy（不使用 cache）**。

3. **Vercel `JSON5: invalid character 'h' at 2:9`（精簡 `vercel.json` 後仍失敗）**  
   - 現象：CLI 一載入專案設定即失敗，未進入 `pnpm install`。  
   - 釐清：精簡後之 **`apps/nx-ui/vercel.json` 第 2 行為 `"framework": "nextjs"`，第 9 欄為字母 `w` 而非 `h`**，故錯誤較可能來自 **Vercel 將「儀表板 Production 覆寫／舊部署設定」與檔案合併後的內容**，而非單一檔案字面。先前推測僅因 `$schema` 的 `https://` 觸發註解誤判，**無法解釋**拿掉 `$schema` 後仍同錯誤，故予以修正敘述。  
   - 處理：**刪除 repo 內 `apps/nx-ui/vercel.json`**，避免與後台設定重複合併；請在 Vercel **Project → Settings → Build & Deployment** 手動設定 **Install Command**：`pnpm --dir ../.. install --frozen-lockfile`（Vercel 仍會先 `cd` 至 Root Directory，再執行此指令）。並 **關閉／清除** Build Command、Output Directory、Framework 等覆寫，改回 **Framework Preset：Next.js**、預設 build（`next build`）。  

4. **Monorepo 單一 pnpm lockfile**  
   - **刪除** `apps/nx-ui/pnpm-lock.yaml`、`apps/nx-ui/pnpm-workspace.yaml`（避免子目錄自成一套 lock／workspace，與根目錄 `pnpm-lock.yaml` 脫節）。  
   - 將原 nx-ui 內 **`ignoredBuiltDependencies`（sharp、unrs-resolver）** 併入 **根目錄 `pnpm-workspace.yaml`**，行為與本機一致。

5. **`JSON5: invalid character 'h' at 2:9` 追蹤**  
   - **`fac54dc`**（無 repo `vercel.json`）仍錯：指向 **Vercel 後台持久化／合併設定**，非僅 Git 檔案。  
   - **`697c5f0`**（**單行** `vercel.json` 僅 `installCommand`、新專案部署）**仍**同一錯誤：檔案內容僅一行 JSON、無 `https`／`//`，**第 2 行不應出現字元 `h`**，故判定為 **只要存在 `apps/nx-ui/vercel.json`，CLI 在解析／合併階段即觸發 bug 或壞合併**。  
   - **處理**：**刪除 `apps/nx-ui/vercel.json`**。本機已驗證在 **`apps/nx-ui` 目錄執行** `pnpm install --frozen-lockfile` 會向上找到 monorepo 根並 **Scope: all 4 workspace projects**，故 Vercel 預設 **Root Directory = `apps/nx-ui` + 預設 `pnpm install`** 即可，**不必**在檔內寫 `installCommand`。後台 **Install Command 覆寫請維持關閉**；其餘覆寫仍應清空（見第 3 點）。

6. **Vercel `JSON5: invalid character 'h' at 2:9` 真正根因**（`36f2c49` 仍錯之後釐清）  
   - 錯誤**與** `apps/nx-ui/vercel.json` **無關**；來自 **repo 根目錄 `turbo.json` 第 2 行**：鍵名誤為 `\"`（應為 **`$schema`**），導致 JSON5 解析在 **`https://…` 的 `h`（第 2 行第 9 欄）** 報錯。Monorepo 啟用「根目錄外檔案」時，Vercel 會讀到根目錄 `turbo.json`。  
   - 已改為：`"$schema": "https://turbo.build/schema.json"`。
