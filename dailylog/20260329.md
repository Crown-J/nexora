     - **不在 repo 內放 `apps/nx-ui/vercel.json`**（見下方第 3 點）：改由 Vercel 專案設定處理 monorepo 安裝指令。  
   - Vercel 後台建議再確認：**Root Directory** = `apps/nx-ui`、**Include files outside the root directory** 開啟；**勿**自訂 **Output Directory**；**勿**保留舊版 Production 覆寫（`Framework: Other`、`Output Directory: apps/nx-ui/.next` 等）；若仍異常可 **Redeploy（不使用 cache）**。

3. **Vercel `JSON5: invalid character 'h' at 2:9`（精簡 `vercel.json` 後仍失敗）**  
   - 現象：CLI 一載入專案設定即失敗，未進入 `pnpm install`。  
   - 釐清：精簡後之 **`apps/nx-ui/vercel.json` 第 2 行為 `"framework": "nextjs"`，第 9 欄為字母 `w` 而非 `h`**，故錯誤較可能來自 **Vercel 將「儀表板 Production 覆寫／舊部署設定」與檔案合併後的內容**，而非單一檔案字面。先前推測僅因 `$schema` 的 `https://` 觸發註解誤判，**無法解釋**拿掉 `$schema` 後仍同錯誤，故予以修正敘述。  
   - 處理：**刪除 repo 內 `apps/nx-ui/vercel.json`**，避免與後台設定重複合併；請在 Vercel **Project → Settings → Build & Deployment** 手動設定 **Install Command**：`pnpm --dir ../.. install --frozen-lockfile`（Vercel 仍會先 `cd` 至 Root Directory，再執行此指令）。並 **關閉／清除** Build Command、Output Directory、Framework 等覆寫，改回 **Framework Preset：Next.js**、預設 build（`next build`）。

5. **`JSON5: invalid character 'h' at 2:9` 追蹤**

6. **Vercel `JSON5: invalid character 'h' at 2:9` 真正根因**（`36f2c49` 仍錯之後釐清）

## 討論與決策（Q&A）

### `NEXT_PUBLIC_NEXORA_RUN_MODE`：怎麼用

| 模式 | `.env.local`（本機）或 Vercel Environment Variables | 行為 |
|------|-----------------------------------------------------|------|
| **development** | `NEXT_PUBLIC_NEXORA_RUN_MODE=development` 或**整行省略** | 與先前相同：需 `NEXT_PUBLIC_API_URL`（如 `http://localhost:3001`）、本機跑 `nx-api` + PostgreSQL + migrate/seed，用真帳密登入。 |
| **demo** | `NEXT_PUBLIC_NEXORA_RUN_MODE=demo` | 登入頁**任意帳號＋密碼**（欄位仍須非空）即可進 `/dashboard`；**不**打後端。其他仍呼叫 API 的頁面可能顯示錯誤，可搭配既有 mock 資料頁面展示。 |

**注意**：`NEXT_PUBLIC_*` 在 Next 為**建置時**嵌入；改變數後需 **`pnpm dev` 重啟** 或 **Vercel Redeploy**。正式產品網域請勿設 `demo`。

**Vercel 建議**：可開**第二個專案**或 Preview 分支專用 demo：`NEXT_PUBLIC_NEXORA_RUN_MODE=demo`；開發用專案維持 `development` 或不設。

**登出**：會清除 access token 與 demo 用的 `sessionStorage` 鍵，回到登入頁。
